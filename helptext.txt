Usage: unstructured-ingest [OPTIONS]

Options:
  --max-docs INTEGER              If specified, process at most specified
                                  number of documents.
  --flatten-metadata              Results in flattened json elements.
                                  Specifically, the metadata key values are
                                  brought to the top-level of the element, and
                                  the `metadata` key itself is removed.
  --fields-include TEXT           If set, include the specified top-level
                                  fields in an element. Default is
                                  `element_id,text,type,metadata`.
  --metadata-include TEXT         If set, include the specified metadata
                                  fields if they exist and drop all other
                                  fields. Usage: provide a single string with
                                  comma separated values. Example: --metadata-
                                  include filename,page_number
  --metadata-exclude TEXT         If set, drop the specified metadata fields
                                  if they exist. Usage: provide a single
                                  string with comma separated values. Example:
                                  --metadata-exclude filename,page_number
  --partition-by-api              Use a remote API to partition the files.
                                  Otherwise, use the function from
                                  partition.auto
  --partition-endpoint TEXT       If partitioning via api, use the following
                                  host. Default: https://api.unstructured.io/g
                                  eneral/v0/general
  --partition-strategy TEXT       The method that will be used to process the
                                  documents. Default: auto. Other strategies
                                  include `fast` and `hi_res`.
  --local-input-path TEXT         Path to the location in the local file
                                  system that will be processed.
  --local-recursive               Support recursive local file processing.
  --local-file-glob TEXT          A comma-separated list of file globs to
                                  limit which types of local files are
                                  accepted, e.g. '*.html,*.txt'
  --remote-url TEXT               Remote fsspec URL formatted as
                                  `protocol://dir/path`, it can contain both a
                                  directory or a single file. Supported
                                  protocols are: `gcs`, `gs`, `s3`, `s3a`,
                                  `abfs` and `az`.
  --s3-anonymous                  Connect to s3 without local AWS credentials.
  --azure-account-name TEXT       Azure Blob Storage or DataLake account name.
  --azure-account-key TEXT        Azure Blob Storage or DataLake account key
                                  (not required if `azure_account_name` is
                                  public).
  --azure-connection-string TEXT  Azure Blob Storage or DataLake connection
                                  string.
  --drive-id TEXT                 Google Drive File or Folder ID.
  --drive-service-account-key TEXT
                                  Path to the Google Drive service account
                                  json file.
  --drive-recursive               Recursively download files in folders from
                                  the Google Drive ID, otherwise stop at the
                                  files in provided folder level.
  --drive-extension TEXT          Filters the files to be processed based on
                                  extension e.g. .jpg, .docx, etc.
  --biomed-path TEXT              PMC Open Access FTP Directory Path.
  --biomed-api-id TEXT            ID parameter for OA Web Service API.
  --biomed-api-from TEXT          From parameter for OA Web Service API.
  --biomed-api-until TEXT         Until parameter for OA Web Service API.
  --wikipedia-page-title TEXT     Title of a Wikipedia page, e.g. "Open source
                                  software".
  --wikipedia-auto-suggest BOOLEAN
                                  Whether to automatically suggest a page if
                                  the exact page was not found. Set to False
                                  if the wrong Wikipedia page is fetched.
  --github-url TEXT               URL to GitHub repository, e.g.
                                  "https://github.com/Unstructured-
                                  IO/unstructured", or a repository owner/name
                                  pair, e.g. "Unstructured-IO/unstructured"
  --gitlab-url TEXT               URL to GitLab repository, e.g.
                                  "https://gitlab.com/gitlab-com/content-
                                  sites/docsy-gitlab", or a repository path,
                                  e.g. "gitlab-com/content-sites/docsy-gitlab"
  --git-access-token TEXT         A GitHub or GitLab access token, see
                                  https://docs.github.com/en/authentication
                                  or https://docs.gitlab.com/ee/api/rest/index
                                  .html#personalprojectgroup-access-tokens
  --git-branch TEXT               The branch for which to fetch files from. If
                                  not given, the default repository branch is
                                  used.
  --git-file-glob TEXT            A comma-separated list of file globs to
                                  limit which types of files are accepted,
                                  e.g. '*.html,*.txt'
  --subreddit-name TEXT           The name of a subreddit, without the "r\",
                                  e.g. "machinelearning"
  --reddit-client-id TEXT         The client ID, see https://praw.readthedocs.
                                  io/en/stable/getting_started/quick_start.htm
                                  l#prerequisites for more information.
  --reddit-client-secret TEXT     The client secret, see https://praw.readthed
                                  ocs.io/en/stable/getting_started/quick_start
                                  .html#prerequisites for more information.
  --reddit-user-agent TEXT        The user agent to use on the Reddit API, see
                                  https://praw.readthedocs.io/en/stable/gettin
                                  g_started/quick_start.html#prerequisites for
                                  more information.
  --reddit-search-query TEXT      If set, return posts using this query.
                                  Otherwise, use hot posts.
  --reddit-num-posts INTEGER      The number of posts to fetch.
  --re-download / --no-re-download
                                  Re-download files even if they are already
                                  present in --download-dir.
  --download-only                 Download any files that are not already
                                  present in either --download-dir or the
                                  default download ~/.cache/... location in
                                  case --download-dir is not specified and
                                  skip processing them through unstructured.
  --slack-channels TEXT           Comma separated list of Slack channel IDs to
                                  pull messages from, can be a public or
                                  private channel
  --slack-token TEXT              Bot token used to access Slack API, must
                                  have channels:history scope for the bot user
  --start-date TEXT               Start date/time in formats YYYY-MM-DD or
                                  YYYY-MM-DDTHH:MM:SS or YYYY-MM-DD+HH:MM:SS
                                  or YYYY-MM-DDTHH:MM:SStz
  --end-date TEXT                 End date/time in formats YYYY-MM-DD or YYYY-
                                  MM-DDTHH:MM:SS or YYYY-MM-DD+HH:MM:SS or
                                  YYYY-MM-DDTHH:MM:SStz
  --discord-channels TEXT         A comma separated list of discord channel
                                  ids to ingest from.
  --discord-token TEXT            Bot token used to access Discord API, must
                                  have READ_MESSAGE_HISTORY scope for the bot
                                  user
  --discord-period TEXT           Number of days to go back in the history of
                                  discord channels, must be an number
  --download-dir TEXT             Where files are downloaded to, defaults to
                                  `$HOME/.cache/unstructured/ingest/<SHA256>`.
  --preserve-downloads            Preserve downloaded files. Otherwise each
                                  file is removed after being processed
                                  successfully.
  --structured-output-dir TEXT    Where to place structured output .json
                                  files.
  --reprocess                     Reprocess a downloaded file even if the
                                  relevant structured output .json file in
                                  --structured-output-dir already exists.
  --num-processes INTEGER         Number of parallel processes to process docs
                                  in.  [default: 2]
  --recursive                     Will recurse directories for S3, GCS and
                                  Azure storage.
  -v, --verbose
  --help                          Show this message and exit.
